Case Study: Drone Show Simulation

Motivation / The "Why"

Modern drone research often requires two things that don't mix well: high-level formation planning and low-level manual control. Existing tools usually focus on one or the other. This project bridges that gap, creating a single platform to rapidly test formation algorithms while still letting the researcher manually override or fine-tune individual drones.

What I Did

  • Web Interface: I built a Flask-based web UI that serves as the mission control, allowing for both formation-based and manual control.
  • Image-to-Formation Pipeline: I developed a computer vision pipeline using OpenCV that lets a user upload a 2D image (e.g., a triangle). My code processes the image, identifies the key contour points, and maps them to a set of 3D GPS waypoints for the drone swarm.
  • Simulation Backend: I integrated the PX4 Autopilot SITL (Software-in-the-Loop) with the Gazebo 3D simulation environment to create a realistic physics sandbox.
